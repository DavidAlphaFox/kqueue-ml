kevent is used to register events with kqueue, and fetch events that are available to read from the kqueue. The changelist array contains the new events that need to be registered. The items in the changelist are processed before any new items are read from a kqueue. If eventlist is non empty, the kevent call will block indefinitely till there are n events available to read, n being the length of the eventlist array. The optional timeout parameter can be used instead to specify the maximum interval that kevent should wait for before returning.
